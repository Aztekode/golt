Lightweight backend runtime

Run TypeScript with Go-powered backend primitives.

Golt bundles a TypeScript entry file with esbuild and executes it inside a Go-hosted JavaScript engine. The runtime exposes a small, explicit API for HTTP, SQL, filesystem access, crypto, JWT, environment variables, logging, and outbound requests.

Overview

What is Golt?

Golt is a small TypeScript/JavaScript backend runtime written in Go. It is not trying to clone Node.js. Instead, it gives your scripts a deliberate set of backend APIs that are easy to reason about and evolve.

Explicit runtime

Only the APIs exposed by Golt are available globally. This keeps the runtime surface predictable.

Simple workflow

Create a project, write app.ts, and run it through golt run app.ts.

Backend-focused

HTTP routing, middleware, SQL, file IO, fetch, bcrypt, JWT, and logging are available from the runtime.

app.ts 
const app = Golt.App();

app.use(Golt.logger({ format: "dev" }));

app.get("/", (ctx) => {
  ctx.Json({
    message: "Hello from Golt!",
    runtime: "golt",
  });
});

app.serve(3000);
Quickstart

From zero to a running server.

The current scaffold is intentionally clean: the CLI creates the project files, while the VS Code extension manages editor typings when it detects golt.json.

Golt bundles and executes the TypeScript entry file.

Editor integration

VS Code typings are extension-managed.

When the extension finds golt.json, it can generate the Golt typings under .golt/types and patch tsconfig.json or jsconfig.json. This keeps the CLI scaffold smaller and avoids coupling runtime releases to editor-only type updates.

⚡
Recommended policy: changes that only affect TypeScript editor types should be released through golt-vscode. Runtime behavior should remain in the Go CLI/runtime.
HTTP

HTTP server with routes, middleware, static files, and safe finalization.

Golt.App() creates an HTTP app backed by Go's net/http. Routes use method-aware patterns like GET /users/{id}, and handlers receive a Context object.

HTTP example 
const app = Golt.App();

app.use(Golt.logger({ format: "dev" }));

app.get("/", (ctx) => {
  ctx.Send("Hello from Golt HTTP");
});

app.get("/users/{id}", (ctx) => {
  ctx.Json({ id: ctx.Param("id") });
});

app.post("/users", (ctx) => {
  const body = ctx.ValidateBody({ name: "string" });
  if (!body) return;

  ctx.Status(201).Json({
    ok: true,
    name: body.name,
  });
});

app.static("/public", "./public", true);

app.notFound((ctx) => {
  ctx.Status(404).Send("not found");
});

app.serve(3000);

Database

Use exec() for writes and query() for rows.

Golt wraps Go's database/sql. The API separates commands that mutate state from commands that return rows.

db.exec()

Use for schema changes, inserts, updates, deletes, and other statements that do not return rows.

db.query()

Use for SELECT statements and operations that return tabular results.

SQLite example 
const db = Golt.db.connect("sqlite", "./app.db");

await db.exec(`
  CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL
  )
`);

const insert = await db.exec(
  "INSERT INTO users(name) VALUES (?)",
  "Ada"
);

console.log("Insert result:", insert);

const users = await db.query<{ id: number; name: string }>(
  "SELECT id, name FROM users"
);

console.log(users);

db.close();
Outbound HTTP

Minimal global fetch().

Golt exposes a small fetch() implementation powered by Go's net/http. It supports method, headers, body, timeout, and response helpers.

Fetch example 
const response = await fetch("https://httpbin.org/json", {
  method: "GET",
  timeout: 15000,
});

if (!response.ok) {
  console.log("Request failed:", response.status);
}

const data = await response.json();
console.log(data);
File system

Simple read/write helpers.

Golt.fs currently provides synchronous helpers for small file operations.

File system example 
Golt.fs.writeFile("./hello.txt", "Hello from Golt\n");

const content = Golt.fs.readFile("./hello.txt");

console.log(content);
Security helpers

Password hashing and JWT helpers.

Golt.crypto provides bcrypt hashing and comparison. Golt.jwt provides HS256 token signing and verification.

Crypto + JWT 
const hash = await Golt.crypto.hash("password123");
const isValid = await Golt.crypto.compare("password123", hash);

console.log("Password match:", isValid);

const token = Golt.jwt.sign({ sub: "user_123" }, "secret", 24);
const payload = Golt.jwt.verify(token, "secret");

console.log(payload);
Environment

Access host environment variables.

Golt.env exposes a map of environment variables from the host process.

Environment variables 
console.log("PATH =", Golt.env["PATH"]);
console.log("NODE_ENV =", Golt.env["NODE_ENV"]);
CLI reference

Commands.

The CLI keeps the workflow focused: create a project, run an entry file, or restart on changes.

golt init <project-name>

Creates a new project folder with app.ts, golt.json, and .vscode/extensions.json.

golt run <filename.ts>

Bundles the entry file with esbuild and runs it in the Golt runtime.

golt watch <filename.ts>

Runs the entry file and restarts on TypeScript/JavaScript changes.

Runtime API

Current API surface.

These are the runtime primitives currently documented by Golt.

console.log(...args)

Prints exported JS values to stdout.

Golt.App()

Creates an HTTP app with routes, middleware, static files, and server start.

Golt.logger(config?)

Creates HTTP logging middleware. Formats: dev, tiny, json.

Golt.db.connect(dialect, dsn)

Connects to sqlite, postgres, mysql, or sqlserver.

db.exec(sql, ...args)

Executes non-row SQL statements and returns metadata.

db.query(sql, ...args)

Executes SQL queries that return rows.

fetch(url, options?)

Performs outbound HTTP requests.

Golt.fs

Reads and writes files.

Golt.crypto

Hashes and compares passwords using bcrypt.

Golt.jwt

Signs and verifies HS256 JWT tokens.

Golt.env

Reads environment variables from the host process.

Roadmap

What should come next?

Since HTTP finalization, db.exec(), and the improved init workflow are already in place, the next practical milestone is test coverage and CLI polish that does not force a VS Code extension release.

Near-term

  • Tests for HTTP request finalization.
  • Tests for db.query() and db.exec().
  • Tests for fetch() success and error paths.
  • Recursive watch behavior.

Later

  • Golt.env.get("KEY", fallback).
  • ctx.Body() / ctx.JsonBody().
  • fetch() JSON helpers.
  • Better source maps and original TypeScript line reporting.
