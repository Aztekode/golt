Golt Runtime Documentation

Golt bundles TypeScript/JavaScript (via esbuild) and executes it in a Go-based JS engine with a small, explicit native API surface.

Overview

Golt provides a simple backend scripting workflow: write TypeScript, run it with a single CLI command, and use a small set of runtime globals.

What Golt is

A Go CLI + runtime engine that compiles a TS entrypoint into a single bundle and runs it.

What Golt is not

Full Node.js compatibility. Only the globals explicitly provided by the runtime are available.

Tip: Keep your Golt programs small and explicit—treat the runtime surface like an SDK, not a full OS.

Installation

Install from source (recommended for local development) or via go install.

From source

Windows (PowerShell / cmd) 
go build -o golt.exe .\cmd\golt

Go install

From GitHub module path 
go install github.com/atrox39/golt/cmd/golt@latest

Quick Start

1) Initialize a project

Creates tsconfig + typings + app.ts 
golt init

2) Run your entry file

Windows example 
golt run .\app.ts

CLI

golt init

Creates three files in the current directory:

golt.d.ts

TypeScript global type definitions for the Golt namespace.

tsconfig.json

Baseline TS config: ESNext target/module, strict mode, includes golt.d.ts.

app.ts

Example entrypoint (prints a greeting).

golt run <filename.ts>

Bundles the specified entry file and executes it in the Golt runtime. Compilation errors and runtime exceptions are printed to stdout/stderr.

Note: The current CLI expects a filename argument. Running golt run without a filename may crash.

Runtime API

The runtime surface is intentionally small. Today, these globals are provided by native modules.

console.log(...args)

Prints exported JS values to stdout.

TypeScript 
console.log("Hello from Golt");
console.log({ ok: true, t: Date.now() });

Golt.env

A map of environment variables from the host process. Keys and values are strings.

TypeScript 
console.log("PATH =", Golt.env["PATH"]);
console.log("HOME =", Golt.env["HOME"]);
console.log("CUSTOM =", Golt.env["CUSTOM"]);

Type definitions

Generated by golt init:

golt.d.ts 
/***
* Golt Runtime - TS/JS Backend Engine Global Definitions
*/
declare namespace Golt {
  export const env: Record<string, string | undefined>;
}

Examples

Minimal script

app.ts 
console.log("Hello, Golt!");
console.log("USER =", Golt.env["USERNAME"] ?? Golt.env["USER"]);

Debugging style output

Structured log 
const event = {
  name: "startup",
  ts: new Date().toISOString(),
  pid: Golt.env["PID"] ?? "unknown"
};
console.log(event);

Limitations & Expectations

Not full Node.js

Only the provided runtime globals exist. Treat this as a curated JS runtime, not Node compatibility.

Bundled entrypoint

Execution is based on an esbuild bundle of your entry file, designed for a single-file runtime experience.

Practical guidance: Start by writing scripts that rely on console and Golt.env only. Add native modules as the runtime grows.
