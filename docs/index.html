Go + Volt A Go-native way to run TypeScript

Run TypeScript on the Go. Power TypeScript with Go voltage.

Golt bundles your TypeScript entry file with esbuild and executes it in a Go-based JS engine, with a small, explicit runtime API.

How it works

A clean circuit: compile → run → extend.

Golt is intentionally simple: it builds a bundle from your entry file, spins up a Go-hosted runtime, then registers native modules (like console, Golt.env, and Golt.serve) before executing your program.

API

Small, explicit runtime primitives.

Today, Golt ships three globals. Keep scripts minimal, then grow capabilities through native modules.

console.log(...args)

Print exported JS values to stdout.

console.log("Hello from Golt");
console.log({ ok: true, now: Date.now() });
Golt.env

Access host environment variables: Record<string, string | undefined>.

console.log("PATH =", Golt.env["PATH"]);
console.log("NODE_ENV =", Golt.env["NODE_ENV"]);
Golt.serve(port, handler)

Start an HTTP server and route inside a single handler.

Uses the process-global default HTTP mux with a single handler at / (matches all paths). No built-in router/middleware: every request hits handler(ctx), so branch on ctx.Url() and ctx.Method(). ctx.Send defaults to 200 if no status is set; handler errors respond 500. Avoid calling Golt.serve multiple times for now. 
Golt.serve(3000, (ctx) => {
  if (ctx.Url() === "/" && ctx.Method() === "GET") {
    ctx.Send("hello");
    return;
  }

  ctx.Status(404).Send("not found");
});
HttpContext: Method(), Url(), Status(code), Send(body)
Quickstart

From zero to running in minutes.

golt init scaffolds a minimal TypeScript project (including typings), and golt run executes your entry file.

TypeScript typings

golt init generates golt.d.ts for Golt.env. Add HTTP typings manually if you want editor autocomplete for Golt.serve. 

type HttpContext = {
  Method(): string;
  Url(): string;
  Status(code: number): HttpContext;
  Send(body: string): void;
};

declare namespace Golt {
  export function serve(port: number, handler: (ctx: HttpContext) => void): void;
}
Install

Build from source (Windows example):

go build -o golt.exe .\cmd\golt
Or install with: go install .\cmd\golt
Run

Initialize and execute your first script:

golt init
golt run .\app.ts
Tip: golt init creates golt.d.ts, tsconfig.json, and app.ts.
